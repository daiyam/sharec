Introduction

Sharec ‚Äì new tool, which will solve headache about configuration. Create it once and reuse everywhere.

With that tool you can easily create a really "bloat" configs, pack it into extertal package and install as simple npm-package.

Installation

If you want to install sharec just use your lovely manager:

npm i --save-dev sharec

Or with yarn

yarn add -D sharec

‚ö†Ô∏è Install sharec only in your configs directory!

Configuration

When you have installed sharec you should modify your configuration package.json with:


            "scripts": {
              "postinstall": "sharec"
            }

Then should create configs directory and init npm package in it:


            mkdir configs
            cd ./configs
            npm init -y

After initialization you can install dependencies right here ‚Äì in configs directory.

For example, you need to install eslint. Exectute that commands in configs directory:


            npm i --save-dev eslint
            eslint --init

Answer on some questions and eslint will be ready to share!

Sharing

So, now you can publish your config or just share it with github.

npm publish

Just publish you config and install into your project.


            cd my-awesome-project
            npm install --save-dev my-awesome-config
            npm install

‚ö†Ô∏è You should install new dependencies in "hand" mode!

Configuration was applied! üôå

How it works

Sharec flow is really simple. There are some steps:

  1. Copying configs from config directory
  2. Adding dependencies from config/package.json
  3. Seting inject status as true

If you are already install config it will not be injected into your project and you will see warning message.
